# 贡献指南

感谢您考虑为 Hide My Tracks 项目做出贡献！以下是一些指导原则，帮助您参与项目开发。

## 贡献方式

您可以通过以下方式为项目做出贡献：

1. 报告 Bug
2. 提出新功能建议
3. 提交代码改进
4. 改进文档
5. 分享使用经验

## 提交 Issue

如果您发现了 Bug 或有功能建议，请提交 Issue：

1. 使用清晰的标题描述问题或建议
2. 详细描述问题的复现步骤或新功能的使用场景
3. 提供系统环境信息（操作系统版本、Shell 类型等）
4. 如可能，附上错误日志或截图

## 提交 Pull Request

如果您想直接贡献代码，请按以下步骤操作：

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/amazing-feature`)
3. 提交您的更改 (`git commit -m 'Add some amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 提交 Pull Request

## 代码规范

- 保持 Bash 脚本风格一致
- 为新增功能添加适当的注释
- 确保错误处理机制完善
- 测试您的代码在不同环境下的兼容性

## 功能增强建议

以下是一些可能的改进方向：

- 支持更多类型的日志清理
- 添加选择性清理选项
- 提供更详细的操作日志
- 增加恢复功能或备份机制
- 支持更多 Linux 发行版的特定日志

## 许可证

通过贡献代码，您同意您的贡献将在 MIT 许可证下发布。
